System Design

Design Goals

Primary Objectives

  • Automate cafeteria waste sorting process
  • Minimize contamination between waste streams
  • Ensure safe and reliable operation
  • Process items in real-time

Technical Requirements

  • Object detection with >95% accuracy
  • Processing time under 2 seconds
  • Safe robot movements
  • Reliable item classification

System Architecture

High-Level Architecture

  • Vision System
    • DETR-based object detection
    • Real-time image processing
    • Object classification system
  • Robot Control
    • Inverse kinematics for movement
    • Path planning algorithms
    • Gripper control system
  • Integration Layer
    • ROS communication framework
    • Coordinate transformation system
    • State management

Design Approach

Core Components

At the foundation of our design is a robust client-server architecture. We've deployed our DETR object detection model on a dedicated server at Berkeley's data center, which communicates with a client script running on our Sawyer robot. This separation allows us to leverage powerful computing resources for vision processing while maintaining responsive robot control.

The motion planning system orchestrates the robot's actions with precision. When a bowl is detected, our system calculates the optimal path for grasping and lifting it. The content classification logic then determines whether the items inside are compostable food waste or reusable utensils, directing the robot to the appropriate disposal location. Each movement is carefully planned to ensure smooth, efficient operation.

Safety is woven into every aspect of our design. Real-time workspace monitoring ensures the robot operates safely in a dynamic environment, while our collision avoidance system prevents any unwanted interactions. An emergency stop system stands ready to halt operations if needed, making our system not just efficient but also reliable and safe for cafeteria environments.
