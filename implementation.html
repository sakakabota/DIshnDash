Implementation

A step-by-step breakdown of how Dish-n-Dash works.

Hardware

We used the following hardware components:

  • Sawyer Robot Arm: Executes smooth and precise motions for waste sorting.
  • Intel RealSense Camera: Provides depth and color images for object detection.
  • Custom Sorting Bins: Designed to align with the robot's motion planning system.

Software

The software pipeline integrates the following systems:

  • Object Detection: We used DETR (a transformer-based detection model) with a ResNet-50 backbone to classify objects and identify their centroids.
  • Depth Processing: Depth images from the RealSense Camera were used to convert 2D pixel coordinates into 3D points in the robot's frame.
  • Trajectory Planning: The robot's motion was planned using inverse kinematics to pick up objects and sort them accurately.

Pipeline Overview

Here's an overview of the Dish-n-Dash process:

  1. Capture an image of the waste item using the Intel RealSense Camera.
  2. Process the image through the object detection model to identify the item and its centroid.
  3. Convert the 2D centroid into a 3D goal point using depth data.
  4. Plan a trajectory for the robot to pick up the item using inverse kinematics.
  5. Sort the item into the correct bin based on its classification.

Challenges

Some key challenges we faced during implementation included:

  • Calibrating the robot's motion to ensure precise sorting.
  • Resolving classification errors for certain objects like bananas in unusual orientations.
  • Dealing with hardware limitations such as short cable lengths causing collisions.
